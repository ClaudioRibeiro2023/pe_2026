## 1. Princípios de execução e arquitetura

| Regra | Descrição |
|---|---|
| Iniciativa sem programa não entra | Toda INIT pertence a COM-P1, COM-P2, COM-P3 ou COM-P4 |
| Sem Sponsor + Dono não existe INIT | Iniciativa sem dono não tem execução — não entra no plano |
| Faseamento obrigatório | COM-P1 é pré-condição. COM-P2/P3/P4 ativam com liderança contratada |
| WIP máximo Q2: 3 pesadas | Líder novo não pode absorver tudo ao mesmo tempo |
| Evidência é obrigatória | Entrega sem evidência verificável não conta como concluída |
| Tipo define comportamento | IMPLANTAÇÃO: instala e vira rotina · PROJETO: início/fim · ROTINA: cadência fixa |

### Matriz de responsabilidades

| Papel | Definição |
|---|---|
| **Sponsor** | Garante prioridade, destrava decisões e cobra execução — nível Direção |
| **Dono (Accountable)** | Responde pelo resultado e pela evidência — 1 pessoa, sem delegação |
| **Responsável (Responsible)** | Executa e entrega evidência — pode ser mais de um |
| **Partners** | Áreas necessárias: CS, RH, Financeiro, Operação, P&D, Marketing |
| **Sign-off** | Valida formalmente que a entrega está concluída |

> **Padrão de evidência:** `COM2026/<Programa>/<Iniciativa>/EVID_AAAAMM_descrição.pdf|md|xlsx`

| **Como executar** | (1) Definir questionário (NPS + 3 perguntas abertas). (2) Definir gatilho de envio (após resposta do cliente à proposta). (3) Implementar envio automatizado. (4) Primeira medição Q3. |
| **Critério de pronto** | Pesquisa implantada. Primeira medição realizada em Q3 com ≥ 50% de resposta. |
| **Evidências** | Questionário publicado. Respostas coletadas. Relatório NPS. |
| **Riscos** | Baixa taxa de resposta → pesquisa curta (≤ 2 min). Feedback negativo → ação corretiva no MBR. |
| **Responsabilidades** | Sponsor: Direção · Dono: Líder Comercial · Partners: MKT (pesquisa), CS (relacionamento) · Sign-off: Direção |
