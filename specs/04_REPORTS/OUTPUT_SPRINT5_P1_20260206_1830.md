```
const availablePacks = useMemo(() => {

### 2.2 Periodo Default

```
const DEFAULT_DATE_RANGE: DateRange = { start: '2026-01-01', end: '2026-12-31' }
```

Regra de filtragem por periodo:
- Acoes sem `start_date` E sem `due_date` sao excluidas do range
- Se `start_date > range.end` -> excluida
- Se `due_date < range.start` -> excluida

Hook `useReportData` agora aceita `dateRange`:

```
export function useReportData(packId: string | undefined, dateRange?: DateRange)

Acoes sao filtradas ANTES do calculo de KPIs/status/programa.

## 3. Hardening: Icon Warnings

### Problema

Build emitia warnings do `vite:reporter`:
```
(!) .../download.js is dynamically imported by .../icons.tsx but also statically imported by .../lucide-react.js
```

### Solucao

Migrados **29 arquivos** em `src/features/` de `import { X } from 'lucide-react'` para `import { X } from '@/shared/ui/icons'`.

Apos migracao, apenas 3 arquivos mantem `from 'lucide-react'`:
1. `src/shared/ui/icons.tsx` - o proprio modulo de icones (type import)
2. `src/shared/config/navigation.ts` - type-only import de `LucideProps`
3. `src/types/lucide-icons.d.ts` - type declarations

### Icones adicionados ao modulo compartilhado

`History`, `Edit3`, `FileSearch`, `Boxes`, `Files`, `Archive`, `ImageIcon`, `Tag`, `Wand2`, `ListIcon`, `LayoutGrid`, `Columns3`, `ChevronUp`, `CheckCircle2`, `Save`

### Renomeacoes para evitar conflitos

| Original (lucide) | Shared Icons | Motivo |
|--------------------|-------------|--------|
| `Image` | `ImageIcon` | Conflito com `HTMLImageElement` |
| `List` | `ListIcon` | Conflito com `HTMLOListElement` |
